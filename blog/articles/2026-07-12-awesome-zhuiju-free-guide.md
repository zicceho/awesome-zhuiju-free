# Awesome Zhuiju Free 使用教程：怎么快速找到今天还能用的追剧资源

找追剧资源最烦的地方，往往不是找不到，而是打开之后才发现网站挂了、广告太多，或者根本播不了。

Awesome Zhuiju Free 做的事情很简单：把免费、相对干净的追剧资源整理成一个开源清单，并且每天自动检测一次可用性。你不用挨个试，先看状态再打开，能少踩很多坑。

<!--more-->

推荐优先打开网站版：

- 网站：https://zhuiju.me

如果你想看完整数据、提交反馈或参与维护，可以打开 GitHub：

- GitHub：https://github.com/laoma2053/awesome-zhuiju-free

GitHub 访问不稳定时，可以使用 Gitee 镜像：

- Gitee：https://gitee.com/laoma2053/awesome-zhuiju-free

## 先看资源分类

项目目前按使用场景整理资源，常见分类包括：

- 在线影视：适合直接用浏览器打开观看。
- 影视 APP：适合手机、电视或盒子使用。
- 网盘资源搜索：适合找网盘分享资源。
- 磁力 & BT：适合下载资源。
- 字幕资源：适合找外语片、剧集字幕。
- TVBox 播放器：适合折腾电视端播放方案。
- 影视仓配置地址：适合配合影视仓、TVBox 类应用使用。
- 开源项目：适合想自建、研究或二次开发的用户。

如果你只是想马上看剧，建议先看「在线影视」。如果你在电视或盒子上使用，优先看「影视仓配置地址」和「TVBox 播放器」。

## 看懂可用性状态

每个可检测资源旁边会有状态标记：

| 状态 | 含义 | 建议 |
|---|---|---|
| 🟢 可访问 | 检测时可以正常打开 | 优先尝试 |
| 🟡 访问受限 | 站点存在，但可能有地区、网络或访问条件限制 | 可以尝试，失败就换 |
| 🔴 无法访问 | 检测时打不开或超时 | 暂时跳过 |
| ⚪ 未检测 | 不适合自动检测，或属于开源项目等类型 | 看说明后再决定 |

检测结果由自动任务定期生成，只代表检测时的网络情况。免费影视站本身变化比较快，所以不要把状态理解成永久保证。

## 怎么快速找一个能用的资源

可以按这个顺序操作：

1. 打开 https://zhuiju.me 或 GitHub README。
2. 找到你需要的分类，比如「在线影视」。
3. 优先选择 🟢 可访问的资源。
4. 看资源简介，判断是否符合你的需求。
5. 如果第一个打不开，立刻换下一个，不要在单个站点上浪费时间。

这个项目的价值不是保证某一个站永远可用，而是把一批资源整理出来，并尽量告诉你「今天哪些更值得先试」。

## 如何反馈资源失效

如果你发现某个资源打不开，可以提交失效反馈：

https://github.com/laoma2053/awesome-zhuiju-free/issues/new?template=broken-link.yml

反馈时建议写清楚：

- 资源名称。
- 打不开的链接。
- 你看到的错误现象，比如超时、404、跳广告页。
- 你的访问环境，比如浏览器、手机、电视端、是否需要特殊网络环境。

信息越具体，维护时越容易判断是临时波动、地区限制，还是资源真的失效。

## 如何推荐新资源

如果你知道好用的免费追剧资源，也可以推荐给项目：

https://github.com/laoma2053/awesome-zhuiju-free/issues/new?template=resource.yml

推荐前建议先确认：

- 是否免费可用。
- 是否没有明显弹窗广告或诱导下载。
- 是否有清晰的资源分类。
- 是否能稳定打开。
- 是否适合公开收录。

项目会优先收录免费、干净、体验相对稳定的资源。需要付费、广告过重、风险不清楚的资源，一般不会放进精选清单。

## 常见问题

### 这个项目是不是影视网站？

不是。Awesome Zhuiju Free 是一个开源资源导航和检测清单，本身不存储、不上传影视内容。

### 为什么有些资源需要特殊网络环境？

部分站点的服务器、域名或访问策略可能有地区限制。项目会尽量把这类情况标成「访问受限」，方便你提前判断。

### 为什么不只保留一个最好的资源？

免费追剧资源变化很快，一个站今天可用，不代表明天还可用。保留多个同类资源，并持续检测，实际体验会更稳定。

### 我不会 GitHub，能不能只看网站？

可以。普通用户直接看 https://zhuiju.me 就够了。只有在推荐资源、反馈失效或参与维护时，才需要用到 GitHub。
