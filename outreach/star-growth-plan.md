# 突破 10000 Star 行动方案

> 本文档基于 awesome-zhuiju-free 现状，结合 AI 建议与实操经验整理。目标：从 ~1000 Star 推进到 10000 Star。

---

## 一、先看清楚你的优势

在制定策略前，先明确你已经有什么：

| 已有 | 说明 |
|---|---|
| 独立域名 zhuiju.me | 大多数 awesome-list 没有这个 |
| 每日可用性检测 | 差异化竞争力，资源不是死列表 |
| 结构化数据（resources.json + schema） | 可扩展、可程序化消费 |
| Issue 驱动的贡献流程 | 降低贡献门槛 |
| CONTRIBUTING.md 已完善 | 社区协作准备好了 |
| 涵盖 10 个资源分类 | 目标受众宽，不是窄众项目 |

**核心卖点（每次对外宣传都要反复强调）：**
> 免费 · 无广告 · 开源 · 每日自动检测可用性 · 社区共同维护

## 二、执行优先级矩阵

按「影响力 / 执行成本」分级：

```
高影响 低成本 → 立即做（本周）
高影响 高成本 → 计划做（本月）
低影响 低成本 → 顺手做
低影响 高成本 → 不做
```

## 三、立即执行（本周，不超过 3 天）

### 3.1 优化 README 的"第一屏"

第一屏（不滚动能看到的内容）决定 70% 的去留率。当前 README 的问题：

- 副标题"追剧不踩坑，资源有人测"很好，但**没有一句话解释"为什么这个列表比其他列表好"**
- 缺少一个**表格或清单说明项目亮点**（Features 区块）
- 缺少 Star 徽章（shields.io 的 GitHub Stars 徽章），Star 数本身是最强信任背书

**建议在 README 顶部加入：**

```markdown
## 为什么选这个列表

| | 普通导航站 | awesome-zhuiju-free |
|---|---|---|
| 广告 | 有 | 无 |
| 资源是否失效无人管 | 常见 | 每日自动检测 |
| 是否开源透明 | 否 | 是，数据公开 |
| 能否参与贡献 | 否 | 可以，Issue 直接提交 |
```

以及加上 Stars 徽章：
```markdown
[![GitHub stars](https://img.shields.io/github/stars/laoma2053/awesome-zhuiju-free?style=for-the-badge)](https://github.com/laoma2053/awesome-zhuiju-free/stargazers)
```

### 3.2 补全 GitHub 仓库信息

- **About 描述**（仓库右上角齿轮）：填写一句话，建议："免费无广告追剧资源导航，每日自动检测可用性。TVBox · 影视仓 · IPTV · 在线影视"
- **Topics 标签**：检查是否包含以下全部：
  `tvbox` `iptv` `awesome-list` `movie` `free` `chinese` `streaming` `tvbox-config` `video` `awesome` `影视` `追剧` `无广告`
- **Website 字段**：填入 `https://zhuiju.me`

### 3.3 发布首篇掘金文章

标题方向：《一周近 1000 Star，我是怎么做这个免费追剧开源项目的》

结构：
1. 痛点（找资源踩坑的经历）
2. 项目介绍（截图 + 一句话核心卖点）
3. 核心亮点：每日检测、开源数据、社区驱动
4. 目前收录了什么
5. 如何参与贡献
6. GitHub 地址

> 掘金首发后，同内容可分发到：segmentfault、CSDN、博客园

## 四、本周内完成

### 4.1 V2EX 发帖

同时发两个节点：

- `/go/share`（分享创造）
- `/go/movie`（电影）

发帖要点：
- **不要写广告文**，写"我做了个项目，希望大家帮我看看哪里可以改进"
- V2EX 用户对"无广告 + 开源 + 程序员做的"极度友好
- 第一楼留下你愿意回复讨论的姿态，及时回复每一条评论

标题建议：`做了个免费追剧资源导航，每天自动检测是否可用，开源的，求反馈`

### 4.2 小众软件论坛投稿

网址：https://meta.appinn.net/c/faxian/10

小众软件用户画像与你的项目高度吻合：追求干净工具、反感广告、技术用户。被主站采纳后（概率较高）会产生长期 SEO 流量。

投稿格式按他们论坛规范，重点说明：免费、无广告、每日检测。

### 4.3 知乎精准卡位

搜索这类问题并写专业回答（结尾自然引出项目）：

- "有哪些免费无广告的在线看剧网站？"
- "TVBox 最新接口地址有哪些？"
- "有什么免费追剧的方法？"
- "影视仓有哪些好用的配置地址？"

知乎回答的长尾 SEO 价值极高，一篇好回答可以持续几年带流量。

## 五、两周内完成

### 5.1 吾爱破解发帖

板块：精华区 > 软件资讯 / 资源荐讨

吾爱破解日活极高，且用户与你的目标受众重合度极高（追剧、找资源、TVBox 用户）。发帖排版要整洁，截图清晰，开头点明"开源免费无广告"。

预期效果：极易上精华，精华帖长期展示，持续带流量。

### 5.2 建立用户沉淀渠道

流量引来了，需要一个地方沉淀。建议方案（二选一或都做）：

**方案 A：Telegram 频道**
- 创建一个 Telegram 频道，用于发布"新增资源播报"和"失效资源通知"
- 在 README 里加入频道入口，非技术用户更倾向于 Telegram 订阅
- 成本：极低，每次更新 resources.json 时手动或自动发一条消息

**方案 B：GitHub Releases + Watch 引导**
- 每次新增超过 5 个资源发一次 Release，写更新说明
- README 加入提示："点击右上角 ⭐ Watch → Custom → Releases，订阅资源更新通知"
- 成本：极低，利用 GitHub 原生功能

两者都做成本也不高，Telegram 对非 GitHub 用户更友好。

### 5.3 NodeSeek / HostLoc 发帖

这两个社区聚集了大量 VPS 玩家、TVBox 用户、自建 Emby/Plex 的重度折腾用户，是你的核心受众。

发帖角度：**不要以宣传为目的**，以"寻找合作维护者"或"想收集更多 TVBox 配置地址，有推荐的吗"切入，在帖子中自然介绍项目。

### 5.4 少数派投稿

网址：https://sspai.com/write

少数派的读者是追求效率和干净体验的用户，高度匹配。投稿方向：

《免费无广告，这个开源项目帮你筛好了追剧资源》

重点突出：**每日自动检测可用性**（这是差异化，普通导航站做不到），开源透明。被收录后 SEO 长尾流量显著。

## 六、本月内完成

### 6.1 小红书图文（强烈推荐）

追剧资源类内容在小红书有巨大的非开发者受众。你不需要懂小红书运营，只需：

1. 注册账号，用项目主题做人设
2. 做 3-5 张竖版图片，内容：
   - 封面："2026 免费追剧神器，程序员整理"
   - 内容页：列出几个分类（在线影视 / TVBox / IPTV），每个放 1-2 个典型资源
   - 结尾：二维码或文字引导到 GitHub / zhuiju.me
3. 评论区置顶 GitHub 地址

> 小红书对 GitHub 链接有限制，可以引导到 zhuiju.me 再跳转 GitHub

关键词：免费追剧 / 电视盒子 / TVBox 配置 / 无广告看剧 / 追剧网站

### 6.2 B站视频（TVBox 配置教程）

门槛最低的拍法：录屏 + 解说，2-3 分钟，展示：

1. 打开仓库 / zhuiju.me 网站
2. 找到一个影视仓配置地址
3. 在 TVBox / 影视仓 App 里填入，打开成功

B站搜索"TVBox 配置"的用户极多，视频简介挂上 GitHub 和 zhuiju.me 地址，一旦有播放量就是持续的被动流量。

## 七、持续运营（长期）

### 7.1 定期发布"月度更新"

每月月初，在 GitHub Releases / Telegram / 各平台同步发一篇：

```
本月新增 X 个资源
- 新增：[xxx]（分类）
- 更新：[xxx] 地址变更
- 下架：[xxx]（失效/高风险）
可用性最高的 5 个资源：...
```

这类更新内容很适合被微信公众号、知乎等二次传播，且能激励社区持续贡献。

### 7.2 响应 Issue 的速度

Issue 回复速度是维持社区活跃度的核心指标，也影响 GitHub 对项目的推荐权重。目标：每个新 Issue 在 24 小时内有初步回复。

### 7.3 收录知名开源项目中的相关资源

主动在以下仓库提交 PR，将 awesome-zhuiju-free 加入它们的列表：

| 目标仓库 | 说明 |
|---|---|
| sindresorhus/awesome | awesome 总列表，门槛高但影响力极大 |
| awesome-selfhosted/awesome-selfhosted | 自托管资源，部分品类重叠 |
| search for "awesome-chinese" | 中文资源导航类列表 |
| search for "awesome-iptv" | IPTV 资源聚合类列表 |

> 提交 sindresorhus/awesome 需要仔细阅读其 pull_request_template.md，要求较严格

### 7.4 联系技术博主推荐

在 X (Twitter) 或微博直接私信：

- GitHubDaily（专门推 GitHub 项目，粉丝量大）
- GitHub 每日早报相关账号
- 少数派作者
- 追剧相关博主

推荐直接发给对方一段现成的文案（包含项目亮点 + GitHub 链接），降低对方转发的决策成本。

| Product Hunt | 你的项目是中文内容，英文受众意义不大 |
| Hacker News | 同上，且 HN 用户对追剧资源导航没有需求 |
| Reddit | 可以试但优先级低，中文内容在 Reddit 回报率低 |
| 双语文档 | 你的内容本质是中文资源，做英文版是无效工作 |
| SEO 建站 | zhuiju.me 已有，不需要另起炉灶 |
| 做 App | 成本过高，偏离当前阶段的核心目标 |
